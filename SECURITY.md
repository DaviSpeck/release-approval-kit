# Security Policy

## Supported Scope

NEXO is actively maintained in this repository, with the current public scope focused on the free Markdown-to-PDF workflow.

Security reports are especially useful for:

- API validation issues
- payload handling problems
- PDF generation edge cases
- unsafe content processing
- data exposure risks
- authentication or authorization issues if new protected areas are added

## Reporting a Vulnerability

Please do not open public GitHub issues for sensitive vulnerabilities.

Instead, report them to:

- `support@speck-solutions.com.br`

Include:

- a clear description of the issue
- potential impact
- reproduction steps
- proof-of-concept payloads or screenshots when safe to share
- any suggested mitigation if you have one

## Response Expectations

NEXO will review reports as quickly as possible and prioritize based on impact and exploitability.

The current best-effort targets are:

- initial acknowledgment within 3 business days
- status update after triage
- remediation timeline based on severity and scope

## Safe Disclosure

Please avoid:

- public disclosure before a fix is available
- automated high-volume testing that may affect availability
- accessing, modifying, or exfiltrating data that is not your own

Good-faith reports are appreciated and help improve the project.
