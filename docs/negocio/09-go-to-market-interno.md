# 09. Plano de Go-to-Market Interno

## Introdução para Times
1. Piloto com 1 squad de alta cadência
2. Template padrão para aprovação
3. Ritual semanal de revisão dos pacotes
4. Expansão progressiva por área

## Política de Uso
- Obrigatório para release com impacto em usuário
- Fluxo simplificado para hotfix pequeno
- Acesso por role e retenção definida por workspace

## KPIs de Adoção
- % releases com pacote gerado
- % aprovações em primeiro ciclo
- Tempo médio de preparação do pacote
- Satisfação dos aprovadores (NPS interno)
